Why it’s different

Statistical AI. Deterministic governance. Finally, both.

LLMs are brilliant at intent — and unreliable at structure. Left to parse work breakdowns or compute Earned Value on token probabilities, they drift. Backcast doesn’t let them.

  • —The AI proposes the intent.
  • —A rigid application layer guarantees the structure.
  • —Every action audited, every time.

Under the hood

Depth, on demand.

The essentials above. The architecture below — open only what you need.

Every change to a task, baseline or cost is captured as a historical delta. Reconstruct the full state of a project at any point in time — the archaeological layers, not just today’s surface.

No raw database access. Agents act through the same Application APIs as human users, passing identical RBAC, constraints and audit. The LLM suggests the intent; the application layer guarantees the structure.

Declarative manifests define roles, specialized prompts, permissions and tool mappings — applying Orchestrator–Workers and Reflection patterns directly to project controls.

Plug in legacy ERPs, web search and custom skills through the Model Context Protocol. Agents discover capabilities dynamically, based on their manifest.

Get started

Run it yourself.

Docker is the recommended path. Clone, configure, bring it up.

git clone https://github.com/ogghst/backcast.git && cd backcast
docker network create traefik-public
cd deploy && cp .env.production.example .env.production
docker compose --env-file .env.production up -d --build
