Root Cause

The transformProjectSpotlight() function in frontend/src/features/dashboard/types.ts used a hardcoded Intl.NumberFormat("en-US", { currency: "USD" }) formatter. The backend ProjectSpotlight schema also did not include the project's currency field, so the frontend had no way to know which currency to use.

TD-104 fixed currency across 25+ components but missed this dashboard transformer because it used a local currencyFormatter instance instead of the shared formatCompactCurrency utility.
