## Root cause (confirmed before editing)

The `Project` model (`backend/app/models/domain/project.py`) defines:
- `contract_value: Mapped[Decimal | None]` — the **real stored financial field** (line 52).
- `currency: Mapped[str]` — real stored field (line 55).
- `budget: Decimal | None = None` — a **non-mapped computed attribute** (line 49), populated by the service layer as the sum of CostElement budgets; **often `None`**.

The AI tool serialization hand-built project dicts that exposed the often-`None` computed `budget` but **omitted `contract_value` and `currency`**. `contract_value` appeared nowhere in `backend/app/ai/`. The public schema (`app/models/schemas/project.py:20,51`) already carries it, so the API contract was never the problem — only the AI-facing serialization was.

## Change implemented

Added `contract_value` (Decimal→`float`, `None`-guarded) and `currency` (str) to the four manually-built project dicts, keeping the existing `budget` field in each. Conversion matched existing style exactly.

| File | Function | Location |
|------|----------|----------|
| `backend/app/ai/tools/project_tools.py` | `get_project` | result dict (~L251) + docstring |
| `backend/app/ai/tools/project_tools.py` | `list_projects` | per-item dict (~L153) + docstring |
| `backend/app/ai/tools/context_tools.py` | `get_project_context` | success return (~L149) + docstring |
| `backend/app/ai/tools/context_tools.py` | `get_project_structure` | inner project dict (~L394) + docstring |

**Not touched:** `global_search` (serializes via `response.model_dump()` of a search-hit schema — not a project financial record; out of scope).
