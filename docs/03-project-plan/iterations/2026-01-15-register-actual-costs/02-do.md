## Red-Green-Refactor Execution

### Protocol Summary

**🔴 RED: Write a Failing Test**
1. Write a single, well-named test capturing the simplest required behavior
2. Follow AAA pattern: Arrange-Act-Assert
3. Run test to confirm it fails for the **expected reason**
4. Document why this test matters for acceptance criteria

**🟢 GREEN: Minimal Passing Implementation**
1. Write the **minimum code** necessary to make the test pass
2. Resist adding functionality not required by current tests
3. Keep implementation intentionally simple
4. Run all tests to confirm new test passes and no regressions

**🔵 REFACTOR: Improve Design While Staying Green**
1. Examine production code for improvements
2. Extract methods, rename for clarity, apply patterns
3. Run tests after **each small change**
4. Document significant refactoring decisions

### Test Template (AAA Pattern)

```python
@pytest.mark.asyncio
async def test_create_cost_registration_with_valid_data_has_required_fields(
    db_session: AsyncSession, sample_cost_element: CostElement
) -> None:
    """Test creating a cost registration with valid data.

    Acceptance Criteria:
    - Cost registration created with provided amount, date, description
    - cost_registration_id is set (root ID)
    - Valid time starts at registration_date
    - Transaction time starts now
    """
    # Arrange
    service = CostRegistrationService(db_session)
    registration_in = CostRegistrationCreate(
        cost_element_id=sample_cost_element.cost_element_id,
        amount=Decimal("100.00"),
        registration_date=date(2026, 1, 15),
        description="Equipment rental"
    )
    actor_id = uuid4()

    # Act
    result = await service.create(registration_in, actor_id=actor_id)

    # Assert
    assert result.cost_registration_id is not None
    assert result.amount == Decimal("100.00")
    assert result.registration_date == date(2026, 1, 15)
    assert result.description == "Equipment rental"
    assert result.created_by == actor_id
```
