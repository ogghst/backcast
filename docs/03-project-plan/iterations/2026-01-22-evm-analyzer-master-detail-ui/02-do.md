**Implementation Analysis (BE-FE-001):**

**Critical N+1 Query Fix:**

The most significant performance bottleneck was identified in `_generate_timeseries_points()`:

**Before (N+1 Problem):**
```python
for date in dates:  # 52 dates for 1-year weekly granularity
    pv = await self._get_pv_as_of(...)      # 1 query per date
    ev = await self._get_ev_as_of_date(...)  # 1 query per date
    ac = await self._get_ac_as_of(...)       # 1 query per date
# Total: 52 dates × 3 queries = 156 database queries
```

**After (Batch Queries):**
```python
# Fetch all data upfront (5 queries total):
bac = await self._get_bac_as_of(...)                           # 1 query
cumulative_costs = await cr_service.get_cumulative_costs(...) # 1 aggregated query
progress_entries = await pe_service.get_progress_history(...) # 1 query
cost_element = await ce_service.get_as_of(...)                # 1 query
schedule_baseline = await sb_service.get_as_of(...)           # 1 query

# Process in-memory (no more queries!)
for date in dates:
    pv = calculate_pv_deterministically(...)  # Pure calculation
    ev = lookup_from_pre_fetched_map(...)     # O(1) lookup
    ac = lookup_from_pre_fetched_map(...)     # O(1) lookup
# Total: 5 queries (96.8% reduction!)
```

**Performance Improvement:**
- Query reduction: 156 queries → 5 queries (96.8% reduction)
- Estimated time reduction: From ~3-5 seconds to <500ms for 1-year weekly time-series
- Memory usage: Minimal increase (caches cost and progress data in memory)
