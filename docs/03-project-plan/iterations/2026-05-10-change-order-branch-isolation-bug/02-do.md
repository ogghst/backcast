## Decisions Made

| Decision | Reasoning | Impact |
|----------|-----------|--------|
| Branch injection pattern | Use `data.branch \|\| branch \|\| "main"` to allow explicit override while defaulting to Time Machine context | Ensures backward compatibility while fixing the bug |
| Type assertion usage | Use `(data as { branch?: string }).branch` to access optional branch field | Maintains type safety without changing generic constraints |
| Delete parameter passing | Pass branch as second parameter: `(id, { branch })` | Works with generated API methods that accept optional branch parameter |
| Backward compatibility | JavaScript allows extra parameters, so delete methods without branch support will ignore it | No breaking changes to existing code |

### FE-003: useDelete Branch Injection ✅

**Changes Made:**
- Added `const { branch } = useTimeMachineParams();` to useDelete hook
- Modified mutationFn to pass branch as second parameter: `(id, { branch: branch || "main" })`
- Uses type assertion to handle different API signatures

**Test Results:**
- ✅ T-003: useDelete injects branch parameter
- ✅ T-003: Handles delete methods without branch support

**Special Consideration:**
- CostElements (versioned) support branch parameter in delete
- Projects and WBEs (non-versioned) don't support branch parameter
- JavaScript allows extra parameters, so the implementation is backward compatible
