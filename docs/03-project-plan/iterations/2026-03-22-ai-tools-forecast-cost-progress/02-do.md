## Key Implementation Details

### Service Method Wrapping

All tools follow the pattern: **@ai_tool decorator → service method call → AI-friendly dict conversion**

Example from `get_forecast`:
```python
@ai_tool(
    name="get_forecast",
    description="Get the forecast for a specific cost element...",
    permissions=["forecast-read"],
    category="forecast",
)
async def get_forecast(
    cost_element_id: str,
    context: Annotated[ToolContext, InjectedToolArg] = None,
) -> dict[str, Any]:
    # Log temporal context
    log_temporal_context("get_forecast", context)

    try:
        # Call service method
        forecast = await ForecastService(context.session).get_for_cost_element(
            cost_element_id=UUID(cost_element_id),
            branch=context.branch_name or "main",
        )

        # Convert to AI-friendly format
        result = {
            "id": str(forecast.forecast_id),
            "eac_amount": float(forecast.eac_amount),
            # ...
        }
        return add_temporal_metadata(result, context)
    except Exception as e:
        return add_temporal_metadata({"error": str(e)}, context)
```

### Temporal Context Pattern

All tools use:
1. `log_temporal_context(tool_name, context)` - For observability
2. `add_temporal_metadata(result, context)` - For result transparency

### Error Handling Pattern

All error paths return:
```python
{
    "error": "Human-readable error message",
    # Optional: "_temporal_context": {...}
}
```

## Lessons Learned

1. **BaseTool Invocation Pattern**: Decorated tools are `BaseTool` instances, must call via `.ainvoke()` not directly
2. **Async Fixture Setup (CORRECTED)**: Use `@pytest_asyncio.fixture` for fixtures that async tests depend on in pytest-asyncio strict mode
3. **Temporal Context**: All tools benefit from temporal logging, even non-branchable entities
4. **Service Wrapping**: Clean separation - tools handle conversion, services handle business logic
5. **Decorator Implementation**: @ai_tool decorator stores permissions in `tool._tool_metadata.permissions`, not `tool.permissions`
6. **RBAC Testing**: Integration tests need to mock `get_rbac_service()` at the module level for decorator testing
7. **Fixture Data Management**: Use tuples or dicts to return multiple values from async fixtures for test flexibility
8. **Error Path Coverage**: Edge case testing significantly improves coverage and robustness
