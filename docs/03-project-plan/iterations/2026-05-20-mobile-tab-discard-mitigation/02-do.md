**Decisions Made:**

- **Persister registry over circular import**: Initially tried importing the persister from main.tsx into useAuthStore, but this caused circular dependency issues and test timeouts in existing auth store tests. Switched to a simple registry pattern (setAppPersister/getAppPersister) in queryPersister.ts. This avoids the circular dependency and keeps the persister accessible from any module.
- **Dynamic import in logout**: The logout function uses `await import("../api/queryPersister")` to get getAppPersister(), wrapped in try/catch. This ensures the persister is optional in test environments and avoids circular module initialization.
- **Version alignment**: Upgraded @tanstack/react-query from 5.100.10 to 5.100.11 to match @tanstack/react-query-persist-client@5.100.11, resolving a QueryClient type incompatibility.
- **Test mock type cast**: Used `as unknown as Query` in test helper to work around insufficient type overlap between mock object and the full Query type.
