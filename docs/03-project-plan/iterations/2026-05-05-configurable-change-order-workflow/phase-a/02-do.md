**Decisions Made:**

- **Config table FK uses PK (id), not root UUID (config_id)**: Audit log and child tables reference `co_workflow_config.id` (PK) for FK constraints, while `config_id` serves as the external-facing stable UUID. This follows the EVCS pattern used elsewhere.
- **Constructor injection with optional config_service**: Services accept an optional `config_service` parameter. When not provided, they create a temporary instance. This allows backward compatibility during migration while supporting DI.
- **Sync-to-async migration**: Methods that were previously sync (`get_user_authority_level`, `get_authority_for_impact`, `calculate_sla_deadline`, `_classify_impact_level`, `_map_score_to_impact_level`, `_get_sla_days`) are now async. All callers updated.
- **Migration merges two heads**: The migration declares `down_revision` as a tuple of both head revisions to unify the migration graph.
- **Seed data uses SLAService values (not ChangeOrderService)**: SLAService values (LOW=2, MEDIUM=5, HIGH=10, CRITICAL=15) are used as the seed, resolving the pre-existing inconsistency with ChangeOrderService's values (LOW=3, MEDIUM=5, HIGH=7, CRITICAL=10).

- **SLAService does not support project-scoped config**: Discovered during testing that SLAService._get_sla_days() calls config_service.get_sla_days() without passing project_id. This is documented as a known limitation -- the test verifies the current behavior (global only).
