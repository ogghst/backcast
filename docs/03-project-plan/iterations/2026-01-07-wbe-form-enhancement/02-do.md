# Iteration: Enhance WBE Creation Form (Context Inheritance & UX) - DO

## Implementation Summary

### Backend

1. **Schema Update**: Added `parent_name` to `WBERead` schema in `backend/app/models/schemas/wbe.py`.
2. **Service Logic**:
   - Modified `WBEService` in `backend/app/services/wbe.py` to join with an alias of the `WBE` table to retrieve the parent's name.
   - Updated `get_by_root_id`, `get_by_project`, `get_by_parent`, and `get_wbe_history` to populate the `parent_name` field.
   - Implemented a robust subquery join to ensure the _current_ version of the parent is used for the name.

### Frontend

1. **Modal Enhancement**:
   - Updated `WBEModal.tsx` to accept `parentWbeId` and `parentName` as props.
   - Added a read-only "Parent WBE" field that displays "Project Root" if `parent_wbe_id` is null, or the parent's name if present.
   - The `parent_wbe_id` is now a hidden field in the form, ensuring it is submitted correctly without being editable by the user.
2. **Context Inheritance**:
   - **Project Detail Page**: Passes `parentWbeId={null}` and `parentName="Project Root"`. Creation level defaults to 1.
   - **WBE Detail Page**: Passes the current WBE's ID and Name as the parent context for children. Creation level defaults to `parent.level + 1`.
   - **WBE List**: Updated to handle parent context for both creation and editing.

## Code Changes
