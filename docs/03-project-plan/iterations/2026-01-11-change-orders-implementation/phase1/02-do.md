**Technical Decisions Made:**

1. **Model Naming**: Changed from `ChangeOrderVersion` to `ChangeOrder` (per user request)

   - Table name: `change_orders` (not `change_order_versions`)
   - Model class: `ChangeOrder` (not `ChangeOrderVersion`)

2. **Dual-Identifier Pattern**:

   - `change_order_id` (UUID): EVCS root identifier for versioning/branching
   - `code` (str): Human-readable business ID (e.g., "CO-2026-001")
   - `code` is NOT unique at table level (appears in multiple branches)

3. **Service Overrides**: Had to override `get_current()` and `create_root()` in `ChangeOrderService`
   - Because `BranchableService` auto-generates field name from class name
   - `ChangeOrder` → would generate `changeorder_id` but our field is `change_order_id`
