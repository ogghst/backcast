**Decisions Made:**

- **Use real ToolRuntime dataclass in tests**: MagicMock cannot satisfy Pydantic validation for ToolRuntime (a @dataclass with complex generics). Created `_make_runtime()` helper that constructs real instances with stubbed dependencies.
- **No DRY refactoring of task/atask**: The sync `task()` and async `atask()` functions share duplicated validation logic, but this matches the SDK's pattern exactly. Keeping them separate avoids introducing shared-state bugs and makes the code straightforward to compare against the SDK reference.
- **_EXCLUDED_STATE_KEYS as frozenset**: Used `frozenset` instead of `set` for immutability and hashability, matching SDK semantics where this set is constant.
- **TASK_TOOL_DESCRIPTION and TASK_SYSTEM_PROMPT**: Copied verbatim from the SDK to ensure identical LLM behavior during the migration. These can be customized later without changing the tool logic.
- **Import alias `langchain_create_agent`**: Used alias to avoid name collision between `langchain.agents.create_agent` (the function) and `DeepAgentOrchestrator.create_agent()` (the method). Tests patch `langchain_create_agent` at module level.
- **Eager subagent compilation**: `_build_subagent_dicts()` now compiles each subagent eagerly via `langchain_create_agent()`, storing the compiled graph in the `runnable` key. This differs from the SDK which passes SubAgent dicts to `create_deep_agent()` for lazy compilation. Our approach is simpler and matches how `build_task_tool()` expects `CompiledSubAgent`-like dicts.
- **No general-purpose subagent**: Per plan decision #5, we do NOT add a general-purpose subagent. Backcast's 7 domain subagents are the complete set.
- **TASK_SYSTEM_PROMPT in system_prompt**: Instead of injecting via a middleware `awrap_model_call` hook (as SDK does), we append TASK_SYSTEM_PROMPT directly to the system_prompt. This is simpler and equivalent since Backcast already appends delegation instructions.

**Blockers:**

- **MyPy crash on google module**: Pre-existing project environment issue. `uv run mypy app/` crashes with `AssertionError: Cannot find module for google` on ALL files, not just ours. Not caused by our changes. Verified with `uv run mypy app/ --strict` on the full project.
- **Pre-existing test failures**: 4 tests in `test_system_prompt.py` and `test_interrupt_node_approval.py` fail independently of our changes. Not in our modification scope.

**Test design:**
- Mocks a compiled subagent runnable with both `ainvoke` and `astream_events`
- Verifies `atask()` calls `ainvoke()` with correct state (HumanMessage with description)
- Verifies the subagent has `astream_events` method (required for LangGraph event propagation)
- Verifies `ToolRuntime.stream_writer` is present (the channel for event propagation)
- The test documents the architectural pattern: LangGraph's runtime automatically propagates events from subagent `astream_events` into the parent's event stream when a tool calls `ainvoke()` on a compiled graph

**Helpers added:**
- `_DummyStreamWriter` -- minimal stream writer stub for `ToolRuntime` construction
- `_make_tool_runtime()` -- creates real `ToolRuntime` instances for streaming tests
