## Implementation Notes

### Key Features Implemented

1. **Lazy-Loading Tree**: Root WBEs are fetched on mount, child nodes are loaded on expansion
2. **TimeMachine Integration**: Component respects `as_of`, `branch`, and `mode` from context
3. **Navigation**: Click handlers navigate to WBE detail (`/projects/:projectId/wbes/:wbeId`) and Cost Element detail (`/cost-elements/:id`)
4. **Currency Formatting**: Budget amounts are displayed as Euro currency with proper formatting
5. **Error Handling**: Comprehensive error, loading, and empty states
6. **Type Safety**: Full TypeScript strict mode compliance

### Known Limitations

- The lazy loading implementation uses direct `fetch` calls instead of the TanStack Query hooks for child nodes. This could be improved in a future iteration to use the existing `useWBEs` and `useCostElements` hooks with proper query invalidation.
- The tree expansion state is not persisted across navigation.
- Performance testing has not been conducted for very large hierarchies (>1000 nodes).

### Potential Improvements

- Add virtualization for very large hierarchies
- Implement tree state persistence
- Add search/filter functionality
- Support for bulk operations on selected nodes
