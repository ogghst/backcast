**Decisions Made:**

- **Separate test class**: New async tests placed in `TestFilterToolsByRoleAsyncCacheRefresh` to keep them isolated from existing sync tests in `TestFilterToolsByRole`. This avoids having to update all existing tests during RED phase.
- **caplog as fixture parameter**: T-005 uses `caplog: pytest.LogCaptureFixture` as a method parameter rather than `import caplog`, following pytest conventions.
- **Top-level imports**: `AsyncMock`, `logging`, and `pytest` added at module level to avoid redundant inline imports in each test method.
- **Mock side_effect pattern**: T-001 and T-002 use `side_effect=[None, [...]]` to simulate cache miss on first `_get_cached_permissions()` call, then valid permissions on the second call after refresh.
- **No caller propagation yet (BE-002 scope)**: Only `filter_tools_by_role()` itself was converted to async. Existing sync callers (and old sync tests) will break with `TypeError: coroutine is not iterable`. This is intentional -- BE-003 handles caller propagation.
- **Graceful degradation**: If permissions are still `None` after refresh (e.g., role does not exist), the function falls back to an empty permission set, which denies all permissioned tools but does not crash.
- **compile_subagents() stays sync**: `compile_subagents()` does not call any async functions directly -- it receives already-filtered tools from the caller. No conversion needed.
- **All callers were already in async contexts**: `_chat_impl()` and `_create_deep_agent_graph()` in `agent_service.py` were already `async def`. `create_agent()` and `create_supervisor_graph()` were sync but their only callers are `_create_deep_agent_graph()` which is async. Clean propagation chain.
- **No logic changes**: Only added `await` keywords and converted `def` to `async def`. Zero logic changes.
- **refresh_permissions_cache on mock services (BE-004)**: Added `AsyncMock()` to `_make_rbac_service_mock()` so that cache-miss tests (which trigger `await refresh_permissions_cache()`) don't crash with `TypeError: object MagicMock can't be used in 'await' expression`.
- **Unified RBAC cache population in integration tests (BE-004)**: The `real_rbac_service` fixture in `test_tool_rbac.py` was updated to also populate the unified RBAC service cache using `_cache_permissions()` from rbac.json. This is needed because `filter_tools_by_role()` uses `get_unified_rbac_service()` (separate from the old `get_rbac_service()`), and without a database session, `refresh_permissions_cache()` fails silently, leaving an empty cache. Pre-populating avoids the issue entirely.
