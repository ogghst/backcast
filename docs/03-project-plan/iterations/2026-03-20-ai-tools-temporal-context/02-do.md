## Decisions Made

| Decision | Reasoning | Impact |
| -------- | --------- | ------ |
| Convert undefined to null for as_of (FE) | Time Machine store returns null for "now", but TypeScript allows undefined. Converting to null ensures consistent "now" semantics | Backend receives null instead of undefined for current time |
| Use regex matching for ISO date assertions (FE) | Date.toISOString() includes milliseconds (.000) which varies by browser/runtime. Using regex (/^date/) makes tests more robust | Tests pass across different JavaScript environments |
| Import store getters in hook (FE) | Using getter methods (getSelectedTime, etc.) instead of direct state access ensures we always get current values | Temporal params are always fresh on each sendMessage call |
| Use `datetime | None` for as_of (BE) | Allows querying historical entity state | Service layer can filter by temporal date |
| Use `str | None` for branch_name (BE) | String identifier for branches (not UUID) | Matches temporal query patterns in service layer |
| Use `Literal["merged", "isolated"] | None` for branch_mode (BE) | Controls query behavior for branch isolation | Enables both isolated and merged views |
| Default as_of to None (BE) | Current state is the most common use case | Backward compatible with existing chat |
| Default branch_name to "main" (BE) | Main branch is the default context | Backward compatible with existing chat |
| Default branch_mode to "merged" (BE) | Merged mode shows combined data | Backward compatible with existing chat |
| Use get_as_of instead of get_by_id (BE) | get_as_of supports temporal queries, get_by_id doesn't | Temporal parameters are properly passed to service layer |
| Add temporal context only when material (BE) | Only add to system prompt when branch != "main" or as_of is set | Reduces noise in AI responses for default queries |
| Keep chat() method without temporal params (BE) | Non-streaming chat doesn't need temporal context in this iteration | Simplifies implementation, maintains backward compatibility |
