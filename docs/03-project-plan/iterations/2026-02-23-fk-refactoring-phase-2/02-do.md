## 4. Decisions Made

- **ORM Navigation**: Used `primaryjoin` with `viewonly=True` and explicit `foreign_keys` list. This is the only way to support ORM navigation in bitemporal models where the FK column references a non-unique root ID.
- **Application-Level Integrity**: Enforced existence checks in all relevant `create` methods. For branchable entities, validation explicitly checks the target branch and falls back to `main`.
- **Relationship Fixes**: Ensured `relationship` is imported from `sqlalchemy.orm` in all affected models.
