## Technical Decisions Made

### 1. Raw SQL for Aggregations

Used raw SQL with `text()` for JSONB field access instead of SQLAlchemy ORM for better performance:
- Cleaner extraction of `impact_analysis_results->'kpi_scorecard'->'budget_delta'->>'delta'`
- Avoids complex type casting issues in SQLAlchemy

### 2. Weekly Aggregation for Trend

Used PostgreSQL `date_trunc('week', ...)` for cost trend:
- Provides meaningful time granularity
- Balances detail with chart readability

### 3. CTE for Aging Items

Used `DISTINCT ON` with subquery for efficient latest status lookup:
- PostgreSQL-specific optimization
- Efficiently finds the most recent status change per change order
