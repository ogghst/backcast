**Key Implementation Details:**

1. **Schema Changes:**
   - `ScheduleBaselineCreate.branch` defaults to `"main"` (not configurable)
   - `ScheduleBaselineCreate.control_date` optional (`datetime | None`)
   - `ScheduleBaselineUpdate.branch` optional (`str | None`)
   - `ScheduleBaselineUpdate.control_date` optional (`datetime | None`)
   - `ScheduleBaselineRead.cost_element_id` optional (for baselines without cost element)

2. **Endpoint Changes:**
   - Direct POST (`/api/v1/schedule-baselines`): Extracts from `baseline_in.branch`, `baseline_in.control_date`
   - Direct PUT (`/api/v1/schedule-baselines/{id}`): Extracts from body, excludes from update_data
   - Nested POST (`/{cost_element_id}/schedule-baseline`): Extracts from body dict, enforces `branch = "main"`
   - Nested PUT (`/{cost_element_id}/schedule-baseline/{id}`): Extracts from body dict
   - Forecast PUT (`/{cost_element_id}/forecast`): Removed query params, extracts from body dict
   - All endpoints: Added datetime string conversion for ISO format handling

3. **Documentation Updates:**
   - Added explicit parameter location pattern (Query for GET/DELETE, Body for POST/PUT/PATCH)
   - Documented DELETE exception with rationale (HTTP/1.1 constraints)
   - Updated examples to show new pattern with context in request body

**Decisions Made:**

- **"Create on Main First" Policy**: POST endpoints hardcode `branch = "main"` to enforce baselines can only be created on main branch first
- **Datetime Conversion**: Added string-to-datetime conversion for `control_date` in all endpoints to handle ISO format strings
- **Optional cost_element_id**: Made `cost_element_id` optional in `ScheduleBaselineRead` to support direct endpoint creating baselines without cost element
- **Parameter Exclusion**: Used `model_dump(exclude={"branch", "control_date"})` in PUT endpoints to avoid duplicate parameter errors
- **Pattern Consistency**: Applied same pattern across all endpoints (nested + direct)
