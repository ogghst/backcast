## Technical Details

### Attachment Upload Flow

1. User selects files in MessageInput component
2. Files are passed to `sendMessage(message, title, executionMode, attachments, images)`
3. Upload happens asynchronously:
   - Documents → `/api/v1/ai/chat/upload-file`
   - Images → `/api/v1/ai/chat/upload-image`
4. On successful upload, backend returns file metadata (ID, URL, etc.)
5. WebSocket message is sent with attachment metadata included
6. Backend processes message with attachment context

### Type Definitions

```typescript
// File attachment metadata
interface FileAttachment {
  file_id: string;
  filename: string;
  file_type: string; // MIME type
  file_size: number;
  url: string;
  uploaded_at: string; // ISO datetime
}

// Extended sendMessage signature
sendMessage(
  message: string,
  title?: string,
  executionMode?: "safe" | "standard" | "expert",
  attachments?: File[],  // NEW: document files
  images?: File[]        // NEW: image files
): Promise<void>
```

### Upload API Integration

- **Document Upload**: `uploadDocumentFile(file, config?)`
- **Image Upload**: `uploadImageFile(file, config?)`
- **Batch Upload**: `uploadMultipleFiles(files, config?)`
- **Progress Callback**: `onUploadProgress: ({ loaded, total, percent }) => void`
- **Error Handling**: Catches axios errors and converts to UploadError type

### Backend Schema Alignment

Frontend types align with backend Pydantic schemas:
- `FileAttachment` matches backend `FileAttachment` schema
- `WSChatRequest` extensions match backend `WSChatRequest` schema
- Upload responses match backend `FileUploadResponse` and `ImageUploadResponse`
