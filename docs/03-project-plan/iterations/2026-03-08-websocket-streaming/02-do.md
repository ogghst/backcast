## Bug Fixes

### [BUG-001] WebSocket Premature Closure

- **Problem:** WebSocket connection closed immediately after the first token or before completion.
- **Cause:** React `useEffect` dependency cycle in `ChatInterface.tsx` caused the WebSocket to tear down and rebuild when `currentSessionId` changed.
- **Solution:** Removed `currentSessionId` from effect dependencies by using functional state updates (`setMessages((prev) => ...)`) and internal refs for session tracking.

### [BUG-002] Session ID Integrity (ForeignKeyViolation)

- **Problem:** `ForeignKeyViolationError` when inserting messages.
- **Cause:** Objects created in the same transaction as the WebSocket loop were not committed before subsequent inserts or due to transaction scope issues.
- **Solution:** Added explicit `await db.commit()` after session creation and message insertion within the streaming loop.
