### 1. Backend Fix: Type Casting in FilterParser

**Problem:** Server returned 500 error when filtering WBEs by level:

```
operator does not exist: integer = character varying
```

**Root Cause:** The `FilterParser.build_sqlalchemy_filters` was treating all filter values as strings, even when the database column was an integer.

**Solution:** Enhanced `FilterParser` to inspect SQLAlchemy column types and cast filter values appropriately:

```python
# backend/app/core/filtering.py
try:
    col_type = column.type.python_type

    if col_type is not str:
         casted_values = []
         for v in values:
             if col_type is bool:
                 # Handle boolean strings
                 v_lower = v.lower()
                 if v_lower in ("true", "1", "yes", "on"):
                     casted_values.append(True)
                 elif v_lower in ("false", "0", "no", "off"):
                     casted_values.append(False)
                 else:
                     casted_values.append(col_type(v))
             else:
                 casted_values.append(col_type(v))
         values = casted_values
except (NotImplementedError, ValueError, TypeError):
    # If we can't determine python_type or cast fails, use original string values
    pass
```

### 2. Frontend Fix: Search Parameter Missing

**Problem:** Cost Elements E2E tests timed out waiting for search API responses.

**Root Cause:** The `CostElementManagement` component wasn't passing the `search` parameter from `tableParams` to the API call.

**Solution:** Added `search` to `queryParams`:

```typescript
// frontend/src/pages/financials/CostElementManagement.tsx
const queryParams = useMemo((): CostElementApiParams => {
  const params: CostElementApiParams = {
    pagination: tableParams.pagination,
    sortField: tableParams.sortField,
    sortOrder: tableParams.sortOrder,
    filters: tableParams.filters as
      | Record<string, (string | number | boolean)[] | null>
      | undefined,
    search: tableParams.search, // ← Added this line
    branch: currentBranch,
  };
  // ...
}, [tableParams, currentBranch, wbeId]);
```

### 3. E2E Test Improvements

**Problem:** Tests were failing due to:

1. Pagination hiding newly created test data
2. Race conditions in dropdown selection
3. Missing wait for API responses

**Solutions:**

#### a. Use Search to Isolate Test Data

```typescript
// Instead of assuming items are on page 1, use search to find them
const searchInput = costElementsSection.locator(".ant-input");
await expect(searchInput).toBeVisible();

const searchResponse = page.waitForResponse(
  (resp) =>
    resp.url().includes("/api/v1/cost-elements") &&
    resp.url().includes("search=")
);
await searchInput.fill(searchableName);
await searchResponse;
```

#### b. Stabilize Dropdown Selection

```typescript
// Wait for dropdown to appear and be stable before clicking
await page.click("#cost_element_form_cost_element_type_id");
await page.waitForSelector(".ant-select-dropdown", { state: "visible" });
await page.waitForTimeout(300); // Small delay for stability
await page.click(
  `.ant-select-dropdown .ant-select-item:has-text("${type.code}")`
);
```

#### c. Relax Filter Conditions

```typescript
// Changed from strict match to just checking for filters param
const filterResponse = page.waitForResponse((resp) =>
  resp.url().includes("filters=")
);
```

### 4. Backend Test Fixes

**Problem:** API tests were expecting array responses but receiving `PaginatedResponse` objects.

**Solution:** Updated all API tests to access `.items` property:

```python
# Before
data = response.json()
assert len(data) == 3

# After
data = response.json()
assert len(data["items"]) == 3
```

Also updated pagination parameters from `skip/limit` to `page/per_page`:

```python
# Before
response = await client.get("/api/v1/projects?skip=0&limit=2")

# After
response = await client.get("/api/v1/projects?page=1&per_page=2")
```
